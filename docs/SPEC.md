# 계약서 위험 조항 탐지 AI 제품 명세

## 1. 탐구 주제 및 이론적 배경

### 1.1 주제명

계약서 위험 조항 탐지 AI

### 1.2 주제 선정 이유 및 목적

현대 사회에서 사용자는 취업, 부동산 거래, 프리랜서 업무, 온라인 서비스 가입 등 다양한 상황에서 계약서를 작성하고 체결한다. 그러나 일반 사용자는 법률 용어와 조항의 의미를 정확히 이해하지 못한 상태에서 계약을 체결하는 경우가 많다.

특히 위약금 조항, 자동 연장 조항, 책임 면책 조항, 일방적 해지 조항 등은 사용자에게 금전적 손실이나 분쟁 위험을 만들 수 있다. 전문 법률 상담은 비용과 시간이 필요하기 때문에 모든 계약서를 전문가에게 검토받기는 어렵다.

본 프로젝트의 목적은 사회초년생, 대학생, 프리랜서 등 계약 경험이 부족한 사용자가 계약서의 위험 요소를 사전에 확인할 수 있도록 AI 기반 계약서 위험 조항 탐지 시스템을 개발하는 것이다.

### 1.3 기술적 배경 지식

계약서는 비정형 텍스트 데이터이며, 조항마다 법률적 의미와 문맥이 중요하다. 본 프로젝트의 최종 모델은 BERT 기반 자연어 처리 모델을 활용한다.

BERT는 Transformer 구조를 기반으로 하며 문장의 앞뒤 문맥을 동시에 고려하는 양방향 언어 모델이다. 기존 RNN, LSTM 방식은 문장을 순차적으로 처리하기 때문에 긴 문장에서 단어 간 관계를 파악하는 데 한계가 있다. 반면 BERT는 Self-Attention을 통해 문장 내 모든 단어의 관계를 동시에 분석할 수 있어 법률 문서처럼 문장 구조가 복잡한 텍스트에 유리하다.

Self-Attention의 핵심 수식은 다음과 같다.

```text
Attention(Q, K, V) = softmax((QK^T) / sqrt(d_k))V
```

여기서 `Q(Query)`는 현재 분석 중인 단어 정보, `K(Key)`는 비교 대상 단어 정보, `V(Value)`는 실제 의미 정보를 나타낸다. 이 계산을 통해 모델은 "계약 해지", "위약금", "손해배상", "책임 면제" 등 단어 조합이 특정 위험과 관련되는지 학습할 수 있다.

### 1.4 현장 적용 제안

- 개인 법률 서비스: 근로계약서, 임대차계약서, 프리랜서 계약서의 위험 조항 1차 검토
- 기업 법무 및 인사 부서: 다량의 계약서 사전 검토 자동화
- 전자계약 플랫폼: 계약 체결 전 위험 조항 경고 제공
- 부동산 플랫폼: 임대차 특약 조항의 위험 요소 안내

서비스화 시 개인정보 비식별화, 법률 데이터 업데이트, AI 판단 책임 범위 고지, 전문가 검토 연결이 필요하다.

## 2. 사용자 정의

### 핵심 사용자

- 계약 경험이 적은 대학생, 사회초년생
- 프리랜서 계약을 자주 체결하는 개인
- 소규모 사업자
- 사내 계약 검토를 1차 자동화하려는 실무자

### 사용자 문제

- 계약서의 긴 문장을 읽기 어렵다.
- 위험 조항이 어디인지 빠르게 찾기 어렵다.
- 불리한 표현인지 판단하기 어렵다.
- 전문가에게 물어보기 전 1차로 확인할 수단이 부족하다.

## 3. MVP 기능 범위

### 포함 기능

- 계약서 텍스트 입력
- 텍스트 파일 업로드
- 조항 단위 자동 분리
- 위험 조항 탐지
- 위험 유형 분류
- 조항별 설명 및 확인 질문 제공
- 전체 위험도 점수 산출
- 리포트 다운로드

### 제외 기능

- 실제 법률 자문
- 판례 검색
- 변호사 상담 예약
- PDF, HWP, DOCX 파싱
- 실제 BERT 모델 학습 및 배포
- 사용자 계정 및 서버 저장

## 4. 위험 조항 분류 체계

| 유형 | 설명 | 예시 키워드 |
| --- | --- | --- |
| 과도한 위약금 | 해지 또는 지연 시 과도한 금전 부담 | 위약금, 손해배상, 배상금 |
| 자동 연장 | 별도 고지 없이 계약이 계속 유지 | 자동 연장, 갱신, 통지하지 않는 경우 |
| 책임 면책 | 상대방 책임을 제한하거나 배제 | 책임을 지지 않는다, 면책, 배상하지 않는다 |
| 일방적 변경 | 한쪽 당사자가 조건을 임의로 바꿈 | 사전 통지 없이, 임의로 변경 |
| 일방적 해지 | 특정 당사자만 계약을 종료 가능 | 즉시 해지, 언제든지 해지 |
| 권리 양도 제한 | 대금, 저작권, 결과물 권리의 불리한 이전 | 모든 권리, 양도, 귀속 |
| 지급 지연 | 대금 지급 시점이 불명확하거나 지연 | 검수 후 지급, 사정에 따라 지급 |
| 경업 금지 | 계약 종료 후 활동 제한 | 경업 금지, 동종 업계, 취업 금지 |
| 관할 및 분쟁 | 특정 법원 또는 불리한 분쟁 해결 방식 | 전속 관할, 중재, 관할 법원 |

## 5. 사용자 흐름

1. 사용자가 계약서 텍스트를 붙여넣거나 `.txt` 파일을 업로드한다.
2. 사용자가 계약 유형을 선택한다.
3. 사용자가 분석 버튼을 누른다.
4. 시스템이 계약서를 조항 단위로 분리한다.
5. 시스템이 위험 조항을 탐지하고 유형을 분류한다.
6. 사용자는 위험도 점수, 위험 조항 목록, 확인 질문을 확인한다.
7. 사용자는 리포트를 다운로드한다.

## 6. 안전 및 책임 고지

- 분석 결과는 일반 정보 제공용이다.
- 계약 체결 여부에 대한 최종 판단을 대신하지 않는다.
- 민감한 개인정보는 입력 전 삭제하는 것을 권장한다.
- 높은 위험도 조항이 발견되면 전문가 검토를 권장한다.
- 모델 업데이트 전후 결과가 달라질 수 있음을 고지한다.

## 7. 성공 기준

- 사용자가 1분 안에 계약서 위험 조항을 확인할 수 있다.
- 위험 조항에는 이유와 확인 질문이 함께 표시된다.
- 분석 결과가 법률 자문으로 오해되지 않도록 UI에 명확한 고지가 있다.
- 추후 BERT API로 분석 엔진을 교체해도 UI 구조를 크게 바꾸지 않는다.
